## Roadmap

### Выбор модели

На основе бенчмарка `qwen2.5-coder:1.5b` показывает 100% успех на задаче `simple_ping` за 1.36 сек, потребляя около 1–2 GB VRAM. Это идеально для MVP.  
**Решение:** использовать **qwen2.5-coder:1.5b** в качестве основной модели. Она одновременно выступает и «лёгким» и «тяжёлым» агентом (через разные промпты). При необходимости (если модель не справляется) – переключаемся на `qwen2.5-coder:7b` (Q4) – влезает в 8GB.

### Обновлённый план работ (5 дней, 3 человека)

#### День 0 (10.04 вечер) – Старт
- **Все** – создание репозитория, `docker-compose.yml` (FastAPI + Ollama + Redis опционально).
- **ML** – установка Ollama, скачивание `qwen2.5-coder:1.5b` и `qwen2.5-coder:7b` (для запаса).
- **Бэк** – FastAPI skeleton, эндпоинт `/health`.
- **Фулстэк** – базовая HTML-страница.

#### День 1 (11.04) – Генерация YAML и валидация
- **ML** – промпт для генерации YAML (метаданные) по описанию задачи. Определение JSON Schema (поля: `operation`, `parameters`, `conditions`). Тестирование на 3-5 примерах.
- **Бэк** – эндпоинт `/generate`: вызов LLM через Ollama, парсинг YAML, валидация через `jsonschema`.
- **Фулстэк** – UI: текстовое поле, вывод результата (YAML или ошибка).

#### День 2 (12.04) – ReAct-цикл и диалог
- **ML** – промпт для feedback: при ошибке валидации передавать структурированное сообщение (поле, ожидаемое значение, полученное). Промпт для уточняющего вопроса (если не хватает данных).
- **Бэк** – состояние сессии (attempts, context). Реализация цикла: до 3 попыток, при ошибке → feedback → повторный вызов LLM. Эндпоинт `/ask` для диалога (вопрос → ответ пользователя → обновление контекста).
- **Фулстэк** – UI: отображение попыток, поле для ответа на уточнение.

#### День 3 (13.04) – Шаблоны, генерация кода, песочница
- **Бэк** – 3 Jinja2-шаблона: `file_ops.lua`, `http_request.lua`, `json_parse.lua`. Template selector по полю `operation`. Генерация Lua-кода.
- **Бэк** – статический анализ `luac`, Docker-контейнер с Lua (ограничения: timeout 5с, память 128MB, без сети). Функция запуска и возврат stdout/stderr.
- **Фулстэк** – UI: подсветка синтаксиса, кнопка «Запустить», вывод результата выполнения.

#### День 4 (14.04) – Интеграция, полировка, документация
- **Все** – отладка end-to-end сценариев (5 типовых запросов).
- **Бэк** – логирование, параметры таймаутов.
- **ML** – финальные промпты, проверка VRAM (пик <8GB), заморозка модели.
- **Фулстэк** – `docker-compose.yml` (FastAPI + Ollama), README, C4 диаграмма, запись демо-видео (2-3 минуты), подготовка презентации (7 мин).

#### День 5 (15.04 утро) – Финальное тестирование и сдача
- **Все** – прогон эталонного запроса организаторов, замер VRAM, фиксация.
- **Фулстэк** – пуш финальной версии, проверка инструкции на чистой машине.
- **Сдача** до 12:00.
