# День 3. Шаблоны Lua и безопасное исполнение

## Роль AI-ассистента
Ты отвечаешь за переход от метаданных к исполняемому Lua-коду и его безопасной проверке.

## Цель дня
Сделать production-путь генерации кода:
1. Выбор шаблона по `operation`.
2. Рендер Lua через Jinja2.
3. Синтаксическая проверка.
4. Запуск в sandbox.

## Обязательные задачи
1. Создать 3 Jinja2-шаблона:
   - `file_ops.lua`
   - `http_request.lua`
   - `json_parse.lua`
2. Реализовать строгий template selector:
   - только явный map `operation -> template`
   - ошибка при неизвестной операции
3. Добавить синтаксическую проверку `luac`.
4. Добавить запуск в Docker sandbox:
   - timeout 5 секунд
   - память 128MB
   - без сети
5. Возвращать результат исполнения:
   - status
   - stdout
   - stderr

## Ограничения
1. Никакой генерации Lua в обход шаблонов.
2. Никаких сетевых вызовов из sandbox.
3. Все ошибки должны быть объяснимыми для пользователя.

## Формат результата дня
1. Для каждого из 3 шаблонов есть минимум 1 рабочий сценарий.
2. Код проходит `luac` там, где должен.
3. Ошибки sandbox корректно отдаются в API.

## Definition of Done
1. На валидном YAML система генерирует Lua и исполняет его в sandbox.
2. На невалидном `operation` возвращается явная ошибка selector.
3. На падении исполнения возвращается контролируемый error response.
