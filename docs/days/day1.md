# День 1. Генерация YAML и базовая валидация

## Цель дня
Собрать первый рабочий контур от текстового запроса пользователя до валидного YAML-контракта MWS Octapi.

## Обязательные задачи
1. Поднять FastAPI-каркас с endpoint-ами `/health` и `/generate`.
2. Подключить локальную модель через Ollama (`qwen2.5-coder:1.5b`).
3. Добавить генерацию YAML c жёстким output contract (`operation` + `parameters`).
4. Реализовать безопасный парсинг YAML и валидацию по `mws_operation.schema.json`.
5. Вернуть единый формат ошибок валидации: `field`, `message`, `expected`, `got`, `hint`.
6. Для `array_filter` добавить проверку `condition` через `protocollab.expression`.

## Ограничения
1. Только локальный inference, без внешних AI API.
2. Без enterprise-компонентов (cache, RAG, orchestration).
3. Максимум простая схема, без избыточной сложности в контракте.

## Definition of Done
1. `GET /health` стабильно отвечает 200 и отражает доступность Ollama/модели.
2. `POST /generate` возвращает валидный YAML или структурированную ошибку.
3. Минимум 3 тестовых запроса проходят с ожидаемым результатом.
4. Логи объясняют, что вернула модель и почему ответ принят/отклонён.

## Передача в День 2
1. Актуальный YAML-контракт.
2. Примеры успешных/неуспешных ответов.
3. Список мест, где нужны уточняющие вопросы.
