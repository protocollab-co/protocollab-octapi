# День 1. Генерация YAML и базовая валидация

## Роль AI-ассистента
Ты работаешь как инженер MVP-пайплайна. Цель дня: сделать стабильный путь от текстового запроса до валидного YAML-артефакта через локальную модель.

## Цель дня
Реализовать первый рабочий контур:
1. Принимает запрос на естественном языке.
2. Вызывает локальную модель через Ollama.
3. Возвращает YAML-метаданные.
4. Валидирует YAML по JSON Schema.

## Обязательные задачи
1. Подготовить минимальный FastAPI-каркас с endpoint-ами `/health` и `/generate`.
2. Зафиксировать JSON Schema для артефакта с полями:
   - `operation`
   - `parameters`
   - `conditions`
3. Реализовать вызов Ollama для модели `qwen2.5-coder:1.5b`.
4. Реализовать парсинг YAML и валидацию через `jsonschema`.
5. Добавить единый формат ошибок валидации (какое поле не прошло и почему).

## Ограничения
1. Только локальная модель, без внешних AI API.
2. Код и данные не покидают локальный контур.
3. Никаких enterprise-компонентов (cache, RAG, orchestration) в этом дне.

## Формат результата дня
1. Запущенный backend с рабочим `/health`.
2. `/generate` возвращает либо валидный YAML, либо структурированную ошибку.
3. Минимум 3 тестовых промпта с зафиксированным результатом.

## Definition of Done
1. `GET /health` стабильно отвечает 200.
2. `POST /generate` на типовых запросах отдает YAML c нужными полями.
3. На невалидном ответе возвращается понятная ошибка валидации.
4. Логи позволяют понять, что именно вернула модель и почему ответ принят/отклонен.

## Передача в День 2
Передай:
1. Текущий контракт YAML.
2. Примеры успешных и неуспешных ответов.
3. Список проблем, где модели не хватает контекста (для уточняющих вопросов в День 2).
