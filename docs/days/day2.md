# День 2. ReAct-цикл и диалог уточнений

## Роль AI-ассистента
Ты отвечаешь за управляемую итеративность. Цель дня: чтобы система умела доуточнять вход и исправлять невалидный YAML в ограниченном цикле.

## Цель дня
Расширить контур Дня 1 до интерактивного цикла:
1. До 3 попыток исправления YAML.
2. Структурированный feedback в модель.
3. Endpoint для уточняющего диалога `/ask`.

## Обязательные задачи
1. Ввести session state:
   - `attempts`
   - `context`
   - `last_validation_errors`
2. Реализовать ReAct-цикл:
   - генерация
   - валидация
   - structured feedback
   - повтор до лимита
3. Добавить `/ask`:
   - backend формирует уточняющий вопрос
   - ответ пользователя добавляется в контекст
4. Стандартизировать формат feedback в модель:
   - поле
   - ожидаемое значение
   - полученное значение
   - hint

## Ограничения
1. Максимум 3 попытки.
2. Если не удалось исправить, возвращать контролируемую пользовательскую ошибку.
3. Не подключать heavy-модель автоматически (только ручная опция, если нужна).

## Формат результата дня
1. Сценарий "ошибка -> feedback -> исправление" работает.
2. Сценарий "нужны данные -> уточняющий вопрос -> продолжение" работает.
3. Есть трассировка по попыткам в логах.

## Definition of Done
1. Минимум 2 кейса успешно исправляются в пределах 3 итераций.
2. Минимум 1 кейс проходит через `/ask` с обновлением контекста.
3. На исчерпании лимита попыток приходит предсказуемый ответ.

## Передача в День 3
Передай:
1. Полный контракт session state.
2. Примеры structured feedback.
3. Финальный объект YAML, который подается в selector шаблонов.
