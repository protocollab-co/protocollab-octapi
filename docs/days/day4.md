# День 4. Интеграция, UI и документация

## Роль AI-ассистента
Ты отвечаешь за сборку цельного демо-контура и готовность проекта к показу.

## Цель дня
Объединить backend, шаблоны и sandbox в единый пользовательский поток + подготовить документацию и артефакты.

## Обязательные задачи
1. Собрать минимальный UI:
   - поле запроса
   - блок YAML/ошибки
   - блок уточняющего вопроса
   - блок Lua-кода
   - кнопка запуска
   - вывод stdout/stderr
2. Довести `docker-compose.yml` до one-command старта.
3. Переработать существующий `README.md` в судейскую инструкцию:
   - точный `ollama pull <tag>`
   - параметры `num_ctx=4096`, `num_predict=256`, `batch=1`, `parallel=1`
   - команды запуска и проверки
4. Подготовить MVP-диаграмму (или упростить текущую) в соответствии с реальной реализацией.
5. Прогнать 5 end-to-end сценариев.

## Ограничения
1. Не добавлять новые крупные компоненты в scope.
2. Документация должна соответствовать фактическому поведению системы.
3. UI функциональный и быстрый, без избыточного усложнения.

## Формат результата дня
1. Рабочий демо-путь от ввода задачи до вывода результата исполнения.
2. README позволяет поднять проект без устных пояснений.
3. Диаграмма совпадает с фактическим pipeline.

## Definition of Done
1. Проект поднимается одной командой.
2. Все 5 E2E-кейсов воспроизводятся по инструкции.
3. README и диаграмма не содержат «лишних» неподнятых компонентов.

## Передача в День 5
Передай:
1. Финальный список команд для запуска/демо.
2. Набор эталонных запросов для защиты.
3. Известные ограничения, которые нужно озвучить жюри.
