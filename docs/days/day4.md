# День 4. Интеграция, UI и документация

## Цель дня
Собрать цельный демонстрационный контур и довести документацию до состояния "запуск без автора".

## Обязательные задачи
1. Собрать минимальный UI (ввод запроса, YAML/ошибки, уточняющий вопрос, Lua-код, запуск sandbox, stdout/stderr).
2. Проверить end-to-end путь: запрос → YAML → Lua → `luac` → sandbox.
3. Довести `docker-compose` до one-command старта.
4. Обновить README:
	- запуск
	- `ollama pull qwen2.5-coder:1.5b`
	- параметры модели (`num_ctx=4096`, `num_predict=256`, `batch=1`, `parallel=1`)
	- проверка через curl
5. Подготовить MVP-диаграмму Mermaid по фактической архитектуре.
6. Прогнать минимум 5 E2E-кейсов.

## Интеграция protocollab (обязательно)
1. В README явно указать используемые компоненты: `yaml_serializer`, `jsonschema_validator`, `expression`.
2. В MVP-диаграмме показать отдельный контур `Backend -> protocollab`.
3. В E2E-кейсах включить минимум один сценарий с ошибкой `condition` и её исправлением.

## Ограничения
1. Без добавления новых крупных подсистем.
2. Документация должна совпадать с реальной реализацией.
3. Приоритет: воспроизводимость демо, а не расширение фич.

## Definition of Done
1. Проект поднимается одной командой по README.
2. Все 5 E2E-кейсов воспроизводятся по инструкции.
3. README и диаграмма не содержат "псевдо-компонентов", которых нет в коде.
4. Переданы: список команд для демо, эталонные запросы, известные ограничения.
5. В документации отдельно зафиксирована роль `protocollab` в пайплайне.
