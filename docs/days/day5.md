# День 5. Финальная проверка и сдача

## Роль AI-ассистента
Ты отвечаешь за предсказуемость демонстрации и финальную готовность к сдаче.

## Цель дня
Подтвердить, что решение воспроизводимо, локально, укладывается в ограничения и готово к оценке.

## Обязательные задачи
1. Прогнать эталонный сценарий организаторов.
2. Зафиксировать пик VRAM через `nvidia-smi`.
3. Проверить запуск на чистом окружении по README.
4. Проверить отсутствие внешних AI API в runtime.
5. Подготовить финальный пакет:
   - репозиторий
   - диаграмма
   - демо-видео
   - презентация

## Ограничения
1. Никаких изменений архитектуры в день сдачи.
2. Только bugfix и стабилизация.
3. Любое отклонение от README считается дефектом релиза.

## Формат результата дня
1. Репетиция демо проходит без ручных обходов.
2. Есть зафиксированные метрики по VRAM.
3. Инструкция в README воспроизводится «с нуля».

## Definition of Done
1. Эталонный запрос проходит успешно.
2. Пиковый VRAM не превышает лимит 8GB.
3. Проект можно запустить и показать по README без автора рядом.
4. Все обязательные артефакты готовы и доступны.

## Чеклист сдачи
1. Ссылка на публичный репозиторий актуальна.
2. README полный и точный.
3. C4/MVP диаграмма приложена.
4. Демо-видео записано.
5. Презентация подготовлена.
